Detailed Description

Log events asynchronously.

Author:
Mike Steinert
Date:
2-17-2010

The async appender will collect events sent to it and then dispatch them to all appenders that are attached to it. Multiple appenders may be attached to an async appender.

The async appender uses a separate thread to serve the events in its buffer. You must call g_thread_init() before attempting to configure an async appender.

Async appenders accept the following properties:

  1. blocking
  2. buffer-size

The blocking property determines the behavior of the async appender when its log event buffer is full. If blocking is TRUE then the client will block until there is room in the buffer. Otherwise the client will not block and the log event will be dropped. In non-blocking mode the async appender will keep a summary of all dropped logging events. The default value is TRUE.

The buffer-size property determines how many messages are allowed in the buffer before the client will block. The default value is 128.

Note:
If blocking is FALSE then the value of buffer-size has no effect.
